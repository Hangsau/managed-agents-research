# S1.2: Academic Surveys on Agent Evaluation (2024–2026)

## Overview
This document catalogs academic surveys and position papers addressing the evaluation, benchmarking, and capability assessment of deployed AI agent systems. The scope is 2024–2026 peer-reviewed publications from arXiv, ACL, COLING, KDD, and other venues. Each entry includes the paper's proposed taxonomy and framework-agnostic applicability to agent evaluation regardless of underlying framework choice.

---

### Evaluation and Benchmarking of LLM Agents: A Survey (Mohammadi et al., 2025)

- **Source URL**: [arXiv:2507.21504](https://arxiv.org/abs/2507.21504)
- **Type**: survey
- **Key taxonomy / dimensions proposed**:
  - Evaluation objectives: agent behavior, capabilities, reliability, safety
  - Evaluation process: interaction modes, datasets/benchmarks, metric computation, tooling
- **What evaluation gap they identify**: Current agent evaluation is fragmented across task-specific benchmarks and application domains; enterprise requirements (role-based access, reliability guarantees, dynamic long-horizon scenarios, regulatory compliance) are systematically underexplored.
- **Their proposed solution direction**: A two-dimensional organizing framework (what to evaluate × how to evaluate) that brings clarity to the scattered landscape and identifies future research directions including holistic, realistic, and scalable evaluation methods for real-world deployment.
- **Framework-agnostic relevance (1–5)**: 5. The taxonomy separates evaluation objectives (applicable to any agent architecture) from evaluation processes (implementation-independent). Both dimensions apply regardless of whether agents use symbolic planning, neural generation, or hybrid approaches.
- **Notable quotes**: "bring clarity to the fragmented landscape of agent evaluation"; "enterprise-specific challenges often overlooked in current research."

### Survey on Evaluation of LLM-based Agents (Li et al., 2025)

- **Source URL**: [arXiv:2503.16416](https://arxiv.org/abs/2503.16416)
- **Type**: survey
- **Key taxonomy / dimensions proposed**:
  - Core LLM capabilities (planning, reasoning, tool use)
  - Application-specific benchmarks (web agents, software engineering agents)
  - Generalist agent assessment
  - Benchmark dimensions analysis
  - Developer tools and evaluation frameworks
- **What evaluation gap they identify**: Shift toward realistic and continuously updated benchmarks is underway, but critical gaps remain in cost-efficiency assessment, safety evaluation (robustness under adversarial scenarios), and scalable fine-grained evaluation methods beyond aggregate metrics.
- **Their proposed solution direction**: Five-perspective analysis organizing existing work across capability types, application domains, and benchmark characteristics; identifies evaluation infrastructure (tools, frameworks) as essential future work.
- **Framework-agnostic relevance (1–5)**: 4. The five-perspective structure (capabilities, applications, benchmarks, dimensions, tools) is architecture-independent. Cost-efficiency and safety dimensions apply universally, though specific benchmark datasets may be framework-specific.
- **Notable quotes**: "more realistic, challenging evaluations with continuously updated benchmarks"; "cost-efficiency, safety, and robustness remain underexplored."

### Beyond Task Completion: An Assessment Framework for Evaluating Agentic AI Systems (MontyCloud Inc. Research, 2025)

- **Source URL**: [arXiv:2512.12791](https://arxiv.org/abs/2512.12791)
- **Type**: methodology proposal
- **Key taxonomy / dimensions proposed**:
  - LLM capabilities
  - Memory systems
  - Tool integration
  - Environment interaction
- **What evaluation gap they identify**: Traditional task-completion metrics (binary success/failure) fail to capture behavioral variations of non-deterministic AI systems in production. Real-world deployments reveal "behavioral deviations overlooked by conventional metrics."
- **Their proposed solution direction**: Four-pillar Agent Assessment Framework examining LLMs, Memory, Tools, and Environment as separate dimensions. Moves beyond success/failure toward behavioral characterization across multi-agent, multi-step workflows with integrated tooling and persistent memory.
- **Framework-agnostic relevance (1–5)**: 5. The four-pillar decomposition (LLM + Memory + Tools + Environment) applies to any deployed agent system regardless of architecture. Each pillar is independently assessable and reusable across symbolic, neural, and hybrid systems.
- **Notable quotes**: "evaluation and assessment of LLM agents remain a fundamental challenge"; "behavioral deviations overlooked by conventional metrics" in production environments.

### AgentBench: Evaluating LLMs as Agents (Liu et al., ICLR 2024)

- **Source URL**: [arXiv:2308.03688](https://arxiv.org/abs/2308.03688)
- **Type**: benchmark / measurement methodology
- **Key taxonomy / dimensions proposed**:
  - Reasoning in interactive environments
  - Decision-making quality
  - Long-term planning and task persistence
  - Instruction comprehension and adherence
  - Performance across model families (API-based vs. open-source)
- **What evaluation gap they identify**: Existing benchmarks conflate LLM capability with agent capability. Critical finding: "poor long-term reasoning, decision-making, and instruction following abilities are the main obstacles" for practical LLM agents. Training data composition (e.g., code presence) has ambivalent impacts.
- **Their proposed solution direction**: Multi-dimensional benchmark with 8 distinct interactive environments assessing reasoning and decision-making. Released datasets, environments, and evaluation tools for reproducible agent assessment.
- **Framework-agnostic relevance (1–5)**: 4. The benchmark's interactive environments and multi-dimensional evaluation approach are reusable across frameworks. However, specific environment implementations (e.g., game engines, simulators) may require adaptation.
- **Notable quotes**: "poor long-term reasoning, decision-making, and instruction following abilities are the main obstacles"; "training on code present ambivalent impacts on different agent tasks."

### ADEPTS: A Capability Framework for Human-Centered Agent Design (D'Oro et al., 2025)

- **Source URL**: [arXiv:2507.15885](https://arxiv.org/abs/2507.15885)
- **Type**: capability taxonomy / design framework
- **Key taxonomy / dimensions proposed**:
  - Actuation (ability to take external actions)
  - Disambiguation (ability to seek clarification and resolve ambiguity)
  - Evaluation (ability to assess task progress and outcomes)
  - Personalization (ability to adapt to user preferences and context)
  - Transparency (ability to explain reasoning and decisions)
  - Safety (ability to avoid harm and maintain guardrails)
- **What evaluation gap they identify**: Existing guidance for agent development is scattered: UX heuristics address interface behavior; engineering taxonomies describe internal pipelines; ethics checklists cover governance. No unified user-facing vocabulary exists to specify what agents should minimally do.
- **Their proposed solution direction**: Six-capability framework defining the minimal, user-facing behaviors agents must demonstrate to be understandable, controllable, and trustworthy. Sits at the interface between technical and experience development, applicable across agents (computer use, web navigation, etc.).
- **Framework-agnostic relevance (1–5)**: 5. The six capabilities are intentionally framework-agnostic and user-centered. Applicable to any deployed agent regardless of internal architecture (symbolic, neural, hybrid) or implementation platform.
- **Notable quotes**: "a concise, user-facing vocabulary that tells teams what an agent should fundamentally be able to do"; "sits at the interface between technical and experience development."

### AI Agents vs. Agentic AI: A Conceptual Taxonomy, Applications and Challenges (Seroter et al., 2025)

- **Source URL**: [arXiv:2505.10468](https://arxiv.org/abs/2505.10468) / [ScienceDirect](https://www.sciencedirect.com/science/article/pii/S1566253525006712)
- **Type**: taxonomy / position paper
- **Key taxonomy / dimensions proposed**:
  - AI Agents: modular, single-entity systems for task-specific automation (tool integration, prompt engineering, reasoning enhancements, single-task focus)
  - Agentic AI: multi-agent collaboration, dynamic task decomposition, persistent memory, coordinated autonomy
  - Application domains distinguish the two: Agents → customer support, scheduling, summarization; Agentic → research automation, robotics, medical decision support
  - Shared challenges: hallucination, brittleness (Agents); emergent behavior, coordination failure (Agentic)
- **What evaluation gap they identify**: Current evaluation methodologies conflate two distinct paradigms (single-agent tool use vs. multi-agent orchestration), leading to misaligned assessment methods, architecture choices, and performance expectations.
- **Their proposed solution direction**: Clear conceptual separation with distinct design and evaluation strategies. Proposes ReAct loops, RAG, coordination layers, and causal modeling as paradigm-appropriate solutions.
- **Framework-agnostic relevance (1–5)**: 5. The agent vs. agentic distinction applies across any implementation framework. The taxonomy is orthogonal to LLM family, tool choice, or reasoning approach; it classifies *structural* system design rather than components.
- **Notable quotes**: "AI Agents are modular systems for task-specific automation; Agentic AI is a paradigm marked by multi-agent collaboration and dynamic task decomposition"; "paradigm-market fit: symbolic and hybrid architectures dominate safety-critical applications."

### Agentic Artificial Intelligence: Architectures, Taxonomies, and Evaluation of Large Language Model Agents (2025/2026)

- **Source URL**: [arXiv:2601.12560](https://arxiv.org/abs/2601.12560)
- **Type**: survey / taxonomy proposal
- **Key taxonomy / dimensions proposed**:
  - Six modular components: Perception, Brain, Planning, Action, Tool Use, Collaboration
  - Technological transitions: linear reasoning → native inference-time reasoning; fixed APIs → open standards (MCP, Native Computer Use)
  - Critical failure modes: hallucination in action, infinite loops, prompt injection
  - Application domains: digital OS navigation, embodied robotics, specialized workflows (software engineering, scientific discovery)
- **What evaluation gap they identify**: Evaluation practices must address new failure modes emerging from agentic systems (hallucination in action beyond text generation, infinite loops from planning failures, prompt injection vulnerabilities during tool invocation).
- **Their proposed solution direction**: Unified six-component taxonomy enabling systematic decomposition and evaluation of agent architectures. Addresses transitions from classical reasoning to neuro-symbolic reasoning and standardized interoperability (MCP).
- **Framework-agnostic relevance (1–5)**: 5. The six-component taxonomy is universal. Tool use and planning modules are framework-independent building blocks applicable to symbolic, neural, and hybrid agents. Failure modes (hallucination, loops, injection) are universal evaluation targets.
- **Notable quotes**: "shift from linear reasoning procedures to native inference time reasoning models"; "open standards like MCP and Native Computer Use"; "hallucination in action, infinite loops, and prompt injection" as critical evaluation challenges.

### A Review of Prominent Paradigms for LLM-Based Agents: Tool Use, Planning, and Feedback Learning (Li et al., COLING 2025)

- **Source URL**: [ACL Anthology: 2025.coling-main.652](https://aclanthology.org/2025.coling-main.652/) / [GitHub Repository](https://github.com/xinzhel/LLM-Agent-Survey)
- **Type**: survey / unified taxonomy
- **Key taxonomy / dimensions proposed**:
  - Three paradigms: Tool Use, Planning (including RAG), Feedback Learning
  - Universal components across paradigms: environments/tasks, LLM-profiled roles (policy models, evaluators, dynamic models), common workflows
  - Unified taxonomy addressing inconsistent terminology across frameworks
- **What evaluation gap they identify**: Numerous frameworks exist for each paradigm, but their intricate workflows and inconsistent terminology create confusion. No unified taxonomy spans paradigm boundaries (tool use ↔ planning ↔ feedback learning).
- **Their proposed solution direction**: Unified taxonomy defining shared concepts (environments, LLM roles, workflows) applicable across all three paradigms. Enables systematic comparison and evaluation of frameworks despite implementation diversity.
- **Framework-agnostic relevance (1–5)**: 5. The three-paradigm structure and role-based taxonomy (policy models, evaluators, dynamic models) apply to any LLM-based agent system. Environmental and workflow abstractions are independent of underlying LLM, tool, or data source.
- **Notable quotes**: "intricate workflows and inconsistent taxonomy create challenges in understanding frameworks"; "unified taxonomy to systematically review and discuss frameworks across paradigms."

### Agentic AI: A Comprehensive Survey of Architectures, Applications, and Future Directions (2025–2026)

- **Source URL**: [arXiv:2510.25445](https://arxiv.org/abs/2510.25445) / [Springer Nature: Artificial Intelligence Review](https://link.springer.com/article/10.1007/s10462-025-11422-4)
- **Type**: comprehensive survey / dual-paradigm analysis
- **Key taxonomy / dimensions proposed**:
  - Dual-paradigm framework: Symbolic/Classical (algorithmic planning, persistent state) vs. Neural/Generative (stochastic generation, prompt-driven orchestration)
  - Three cross-cutting dimensions: theoretical foundations/architecture, domain-specific implementations (healthcare, finance, robotics), ethical/governance considerations
  - Paradigm-market fit: Symbolic systems → safety-critical (healthcare, robotics); Neural systems → adaptive, data-rich (finance, education)
- **What evaluation gap they identify**: Current approaches often retrofit classical AI concepts onto neural systems or vice versa ("conceptual retrofitting"). Evaluation frameworks fail to account for paradigm-specific reliability and failure characteristics.
- **Their proposed solution direction**: Structured PRISMA-based dual-paradigm analysis (90 studies, 2018–2025). Proposes intentional hybrid neuro-symbolic integration as future direction, with evaluation frameworks tailored to each paradigm's strengths and failure modes.
- **Framework-agnostic relevance (1–5)**: 4. The symbolic vs. neural distinction is orthogonal to specific implementations but carries strong implications for evaluation methodology (symbolic systems → determinism-focused metrics; neural systems → robustness-focused metrics). Domain-specific findings reduce generalizability.
- **Notable quotes**: "the future of Agentic AI lies not in the dominance of one paradigm, but in their intentional integration"; "paradigm-market fit: symbolic systems dominate safety-critical domains, neural systems prevail in adaptive environments."

### On the Measure of Intelligence (Chollet, 2019) — Background Reference

- **Source URL**: [arXiv:1911.01547](https://arxiv.org/abs/1911.01547)
- **Type**: foundational position paper / measurement theory
- **Key taxonomy / dimensions proposed**:
  - Intelligence as skill-acquisition efficiency (not task performance)
  - Four key components: scope (range of tasks), generalization difficulty (learning curve), priors (initial knowledge), experience (training data/feedback)
  - Abstraction and Reasoning Corpus (ARC) benchmark with priors aligned to innate human cognitive abilities
- **What evaluation gap they identify**: Current benchmarks measure task-specific skill; they fail to distinguish skill acquired through unlimited training data/priors from genuine intelligence (generalization under limited prior knowledge). "Skill is heavily modulated by prior knowledge and experience."
- **Their proposed solution direction**: Define intelligence formally using Algorithmic Information Theory. Propose ARC benchmark designed to minimize advantage of brute-force training and reward efficient learning on novel, abstract reasoning tasks. Shift evaluation from task performance to learning efficiency.
- **Framework-agnostic relevance (1–5)**: 5. Chollet's framework (skill-acquisition efficiency, scope, generalization, priors, experience) is universally applicable to any intelligent system. The distinction between skill and intelligence applies to LLM agents, symbolic agents, embodied agents, etc. ARC's design principles (minimal priors, novel tasks) inform agent evaluation design regardless of implementation.
- **Notable quotes**: "intelligence is skill-acquisition efficiency"; "skill is heavily modulated by prior knowledge and experience"; contemporary AI "still gravitates towards benchmarking intelligence by comparing skill at specific tasks"; ARC designed with "priors aligned to innate human cognitive abilities."

### Analysis

The taxonomy comparison reveals three cross-cutting patterns:

1. **Objective axes**: Recent surveys (2025–2026) converge on four evaluation targets — *behavior* (what the system does), *capability* (what skills it has), *reliability* (consistency and robustness), and *safety* (harm avoidance). Task completion alone (Liu et al., AgentBench) has been superseded by multi-dimensional assessment (Mohammadi, MontyCloud, ADEPTS).

2. **Process axes**: Evaluation methodologies split into two camps: *benchmark-driven* (AgentBench, application-specific suites) and *framework-driven* (Mohammadi's two-dimensional taxonomy, Seroter's paradigm distinction, ADEPTS' six capabilities). The latter enables framework-agnostic evaluation.

3. **Failure modes**: Emerging critical failures (hallucination in action, infinite loops, coordination failure) appear only in 2025–2026 papers, absent from 2024 benchmarks. This signals rapid maturation: as agents move into production (MontyCloud, long-horizon scenarios), evaluation must shift from controlled environments to behavioral robustness in deployment.

Chollet's foundational framework (2019) remains foundational: modern agent evaluation inherits his insight that *efficiency of learning* (not final task performance) is the true measure. ADEPTS and Mohammadi both reframe agent evaluation around *capability demonstration* rather than *task completion*, echoing Chollet's argument that generalization and priors must be explicitly modeled.

## Key Gaps Identified Across All Papers

1. **Production-scale evaluation**: No unified approach for long-horizon, stateful, multi-step agent behavior in production environments (Mohammadi gap; MontyCloud motivation).
2. **Cost-efficiency metrics**: Financial and computational cost assessment largely absent despite emerging in practical deployments (Li et al. gap).
3. **Safety under autonomy**: Adversarial robustness and prompt injection in tool-invocation pipelines remain underspecified (Agentic AI 2601.12560 gap).
4. **Paradigm-appropriate evaluation**: Symbolic and neural agents require different failure-mode detection strategies, but unified benchmarks treat them identically (Agentic AI Survey 2510.25445 gap).
5. **Behavioral non-determinism**: Traditional metrics assume repeatable outcomes; production agents exhibit stochastic behavior, memory state drift, and compounding errors over extended horizons (MontyCloud motivation; AgentBench gap).
