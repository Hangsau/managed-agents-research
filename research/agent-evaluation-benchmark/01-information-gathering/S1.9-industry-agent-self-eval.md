## Industry Vocabulary Clusters

### Autonomy Measures
- **Turn duration** (Anthropic): how long agent operates before human intervention
- **Auto-approval rates** (Anthropic): user acceptance of agent actions without review
- **Interrupt rates** (Anthropic): human intervention frequency
- **Session length** (Replit, GitHub, Anthropic): duration of uninterrupted execution (200+ minutes vs ~45 min typical)
- **Agent-initiated stops** (Anthropic): self-imposed boundaries, reflection points

### Task Complexity Taxonomy
- **Low-to-medium complexity** (GitHub): adds features, fixes bugs, extends tests, refactors code
- **Well-tested codebases** (GitHub, Sourcegraph): implies existing test suite reduces uncertainty
- **Ambiguous requirements** (Devin): acknowledged weakness
- **Architectural decisions** (Devin, Adept): human judgment required
- **Simple vs complex** (Cursor): binary categorization; no quantified threshold

### Benchmark Standards (Open/Proprietary)
- **Open**: OSWorld-Verified (desktop automation, GUI), WebVoyager (real-world web), WebArena (web interaction), SWE-bench (GitHub issues)
- **Proprietary**: Pace insurance (Anthropic), "Dreaming" gains (Anthropic), customer deployment metrics (Devin, Cognition), internal coding benchmark (93 tasks, Anthropic)
- **Adoption pattern**: Anthropic leads in open benchmark use; Devin, Replit, Cursor, GitHub avoid external benchmarks entirely

### Execution Paradigms
- **Synchronous/foreground** (Cursor, Copilot): real-time user interaction, background notification
- **Asynchronous/batch** (GitHub, Copilot Coding Agent, Replit 200-min sessions): overnight PR generation, decoupled from user session
- **Multi-file mutation** (GitHub, Cursor, Anysphere): cross-file edits as success indicator
- **Tool composition** (Adept, Sourcegraph): multi-tool task completion as complexity measure

### Failure Modes (Explicitly Named)
- **Hallucination** (Anysphere support agent): factual invention, false explanations
- **Code breaking** (Replit): previously working code regression
- **Scope creep mid-task** (Devin): requirement changes post-start
- **Complex interfaces** (OpenAI/Operator): slideshow creation, calendar management (not attempted)
- **Vague requirements** (Cursor, GitHub): architecturally ambiguous tasks degrade quality

## Patterns in What Gets Advertised vs Measured

**Benchmark avoidance dominates the agent vendor landscape.** Across 10 vendors surveyed, only Anthropic (Claude Computer Use) and OpenAI (Operator) routinely cite open, third-party benchmarks (OSWorld-Verified, WebVoyager, SWE-bench). The other 8 vendors (Devin, Replit, Cursor, Adept, Sourcegraph, MultiOn, GitHub, Anysphere) rely on either proprietary internal benchmarks, qualitative feature claims, or customer-deployment anecdotes.

**Autonomy claims exceed measured autonomy.** "10x more autonomous" (Replit), "autonomous overnight PRs" (GitHub), "motor cortex for AI" (MultiOn) are high-profile marketing statements. The actual measured autonomy proxies (Anthropic's turn duration ~45 min, GitHub's low-to-medium task scope, Devin's 13.86% SWE-bench) show agents operating within narrow, human-monitored contexts. No vendor publishes sustained multi-hour, zero-human-intervention case studies.

**Task-complexity labeling replaces quantitative thresholds.** "Low-to-medium complexity" (GitHub), "simple vs complex" (Cursor), "4-8 hour junior tasks" (Devin) are boundaries without definitions. This allows marketing claims to remain defensible while measurements remain unmeasurable. Anysphere's "1 billion lines of code daily" similarly lacks denominator context (what % are correct? how many fail?).

**Benchmarks chosen to appear competitively favorable.** OpenAI's 38.1% OSWorld claim is presented as "beats Anthropic," but Anthropic's 72.5–78% range on the same benchmark directly contradicts this. Replit's "3x faster, 10x cheaper" comes from unnamed internal benchmarks never published. Google Mariner's 83.5% WebVoyager is respectable until third-party research (Skyvern) reveals competitors scoring 90%+.

**Failure modes and non-scope are minimized.** Only Devin, Adept, Anthropic, and OpenAI explicitly acknowledge limitations (ambiguous projects, complex interfaces, mid-task scope changes, sensitivity/high-stakes tasks). Replit, Cursor, Sourcegraph, MultiOn, GitHub, and Anysphere publish no capability boundaries.

**Industry splits on evaluation philosophy.** Anthropic+OpenAI favor open benchmarks + measurement infrastructure (turn duration, auto-approval). Devin uses "real-world deployment metrics" but excludes competitors. Vendors without benchmarks (Replit, GitHub, Cursor) substitute feature enumeration (multi-file editing, terminal access, PR generation) as proxy for capability, confusing infrastructure with autonomy.
