# 探索：R²-Mem — Reflective Experience for Memory Search

**延續自**: [[2026-05-20-agent-memory-taxonomy-survey]], [[2026-05-20-aegis-memory-deep-dive]]

## 資料來源

- arXiv:2605.13486v1 [cs.CL] — 2026-05-13
- Title: R2-Mem: Reflective Experience for Memory Search
- Authors: USTC China

## Core Claim

現有 deep memory search agent 把每條 trajectory 當獨立的，無法累積可重複使用的經驗。R²-Mem 在 offline 階段用 Rubric-guided Evaluator 評分歷史 trajectory 的每一步（Planning + Reflection），然後用 self-Reflection Learner 把高分/低分行為蒸餾成「經驗庫」。Online inference 時，從經驗庫取出相關經驗來引導未來的 planning 和 reflection。

## 關鍵設計

### 雙元件框架

1. **Rubric-guided Evaluator**：對 Planning 和 Reflection 兩個 module 各用 4 個維度評分（0-3）
   - Planning: Info Needs Coverage / Non-Redundancy / Tool-Info Alignment / Efficiency
   - Reflection: Sufficiency Judgment Accuracy / Minimal Sufficiency Recognition / Follow-up Query Quality / Answer Completeness Awareness
2. **self-Reflection Learner**：根據評分蒸餾 IF-THEN 形式的經驗
   - 品質閾值：score > K_high → good；score < K_low → bad
   - 經驗分類：Planning experience vs. Reflection experience

### 意外發現：低質量經驗更有價值

Ablation study（Table 4）發現：**只用低質量經驗 > 只用高質量經驗**。失敗案例提供更強的修正信號，比成功案例更有效引導 search robustness。這和直覺相反，但邏輯上合理：高質量行為本來就有效，不需要太多額外引導；低質量行為才是需要被糾正的。

### RL-free

不需要 end-to-end RL。Evaluator 和 Learner 都是 prompt-based，self-reflection 由小模型完成。Table 10 顯示即使不用 GPT-4o 做 evaluator，self-evo 版本仍能持續打敗 GAM baseline，與 GPT-4o-evaluated 版本差距很小（Qwen-14B 上只有 0.43 F1 差距）。

## 與 Aegis Memory 的關係

| | Aegis | R²-Mem |
|--|-------|--------|
| 核心機制 | ACE loop（completion → vote → reflection） | Rubric-eval + experience bank |
| 評估方式 | success/failure signal on memories | Fine-grained rubric scoring per step |
| 經驗來源 | 整條 trajectory 的 vote | Step-level high/low quality diagnosis |
| 儲存形式 | memory with typed edge | IF-THEN experience pairs |
| 適用場景 | Memory-level reflection | Search trajectory refinement |

兩者互補：Aegis 做「記憶層」的 self-improvement，R²-Mem 做「搜尋行為層」的 self-improvement。

## Hermes 啟發

### 可抄襲的設計

1. **Step-level scoring for heartbeat_learning.py**：現有 pattern extraction 是 coarse-grained（整條 trajectory）。可以參考 R²-Mem 的 rubric 設計，把 heartbeat action 拆成 Planning（選什麼 action）/ Reflection（結果如何）/ Efficiency 三個維度評分，而不是只計數。

2. **低質量案例優先原則**：做 ISSUES.md 複盤時，把焦點放在「哪一步做錯了」比「哪一步做對了」更有價值。失敗案例驅動的 reflection 應該是 ISSUES.md 的核心。

3. **Experience bank 形式**：IF-THEN experience pairs 是比 plain pattern 更結構化的知識表示。可以研究用這種形式替代 ISSUES.md 的 free-text root cause 描述。

4. **不依賴強模型的 self-improvement**：Table 10 說明不需要 GPT-4o 也能做到有效的 self-reflection。Hermes 的 self-evolve 不需要等更强的模型才能運作。

### 不適合直接 adopt 的

- 完整的 PGVector + 離線 training 流程（太重）
- 多輪對話追蹤（Hermes 是 stateless cron，trajectory 概念不完全適用）
